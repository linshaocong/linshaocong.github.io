    • github+hexo搭建博客(二)

    今天主要讲怎么使用hexo发博客，首先你肯定要会markdown写文章的方法，这个应该不是很难，不懂的现学一下就好，markdown的软件有很多，自己选择喜欢的就好，我windows系统常用的是Typora 用起来比较顺手，下面进入正题，会讲下原理和一个插件；

    一、发博客原始方法

    1. 你需要明白hexo整个的运行原理是怎么个步骤，我也不是特别确切，但是大体是这样，你的博客文章是存在你本地的博客文件夹>blog>source>_posts 在这个文件夹里面你也应该会发现系统默认的那篇hello-world.md，我们的文章也是放在这里；

    2. 现在我们要发一篇新的，那我们就在这个文件夹下面新建一个markdown文件，也可以通过在文件夹根目录git bash here 打开命令窗口，输入命令new 一个，文件夹就会出现一个新的md文件 代码如下；

    3. 然后我们随便写点在文章里面；

    4. 现在把他传到网上，然后下面三个命令要死记，这是我们每次更新文章要用的；

    5. 我们现在访问一下我们的网站，就能看到我们的那篇文章了；

    二. hexo-admin博客后端管理工具

    前面这种操作麻烦有繁琐，那有没有简单操作？我们可以使用hexo-admin插件；

    1. 首先我们来安装，安装很简单，一句命令，首先我们还是在文件夹里打开git bash here,输入下面的命令；

    2. 我们可以用s命令，把服务启动，本身这个命令，是让我们启动我们的页面在本地，所以你访问 http://localhost:4000/ 是能看到你的网站，这个功能就是为了后面主题修改时候在本地先运行,我们插件的使用就是在这个链接后面接上admin，也就是访问 http://localhost:4000/admin 就可以看到我们的插件页面了，在这个页面我们可以进行新建编辑，可视化的操作，不过不要关掉git bash，不要关掉S服务不然访问不到；

    3. 发布到网页的操作本来可以在插件的deploy实现，不过我没时间去研究，大家可以自己研究，也可以编辑完博客，到git bash 运行那三个命令；
